RSS vs. RSS

One of the strange things about the weblogging world is that there are two XML formats with the same initials: Really Simple Syndication, and RDF Site Summary.

Both ultimately derive from a format from Netscape called RDF Site Summary 0.9 (although it was not technically RDF-compatible). The first revision 0.91 is not backwards compatible, in the sense that an RSS-0.9 document isn’t an RSS-0.91 document:

  • The 0.91 root element is rss rather than rdf:RDF (abandonning the pretence of RDF compatbility).
  • In 0.91, the channel’s item elements are nested inside the channel element (rather than following it).
  • In 0.9 the elements are in the namespace http://my.netscape.com/rdf/simple/0.9/. In 0.91 the elements have no namespace.

Many elements were added based on the Scripting News format, adding per-item metadata beyond the bare headline and URI.

The 0.9x series continues with RSS 0.92 and RSS 2.0, which are upwards-compatible (0.91 documents are 0.92 documents, etc.). RSS 0.92 is the format with the largest number of feeds on the syndic8 web site (as of September 2002).

RSS 1.0 is the product of the RSS-DEV mailing list. It is a successor to RSS 0.9 (retaining the rdf:RDF root element and the positioning of items following the channel), which has had additions made to make it valid RDF, and a system of extension through modules. It is not a successor to 0.91 or 0.92 (with their rss root element etc.), although most of the elements added in 0.92 are also incorporated in 1.0. Neither is it technically upward-compatible with 0.9 (the XML namespace used is different, etc.). Not that this matters much, since 0.9 is no longer deployed.

There was some discussion at the time as to whether the 1.0 format should have a different name. In the end the drivers of both formats each stuck with the initials RSS. Luckily there is a slight tendency to use different file-name extensions (RSS 0.9x tends to use .xml or .rss, whereas RSS 1.0 tends to use .rdf).
